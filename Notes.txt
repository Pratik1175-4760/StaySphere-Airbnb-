STATE:
1. Stateful Protocol: A protocol that maintains state information across multiple interactions or sessions. It allows for the tracking of user interactions and can provide a more personalized experience.
2. Stateless Protocol: A protocol that does not maintain any state information between interactions. Each interaction is treated as independent, and the server does not retain any information about previous interactions. This can lead to simpler and more scalable systems, but may require additional effort to manage user sessions and interactions.

Steps to use npm express-session:
1. Install the express-session package using npm:
   ```
   npm install express-session
   ```
2. Import the express-session module in your application:
    ```javascript
    const session = require('express-session');
    ```
3. Configure the session middleware in your Express application:
    ```javascript
    app.use(session({
      secret: 'your_secret_key',
      resave: false,
      saveUninitialized: true,
      cookie: { secure: true }
    }));
    ```
4. Use the session object to store and retrieve data in your routes:
    ```javascript
    app.get('/set-session', (req, res) => {
      req.session
      .user = 'John Doe';
      res.send('Session data set');
    });
    app.get('/get-session', (req, res) => {
      const user = req.session.user;
      res.send(`Session data: ${user}`);
    });
    ```
5. Start your Express server and test the session functionality:
    ```javascript
    app.listen(3000, () => {
      console.log('Server is running on port 3000');
    });
    ```

Note: This is required in frontend if used in react
import axios from 'axios';
axios.defaults.withCredentials = true;

This tells axios to include cookies in requests, which is necessary for maintaining sessions when making API calls from the frontend to the backend. Make sure to configure your backend to allow cross-origin requests and to set the appropriate CORS headers if your frontend and backend are hosted on different domains.

Note: Make sure to replace 'your_secret_key' with a strong, unique secret key for your application to ensure the security of your sessions.

Steps to use connect-flash:
1. Install the connect-flash package using npm:
   ```
   npm install connect-flash
   ```
2. Import the connect-flash module in your application:
    ```javascript
    const flash = require('connect-flash');
    ```
3. Configure the flash middleware in your Express application:
    ```javascript
    app.use(flash());
    ```
4. Use the flash object to set and retrieve flash messages in your routes:
    ```javascript
    app.get('/set-flash', (req, res) => {
      req.flash('info', 'Flash message set');
      res.send('Flash message set');
    });
    app.get('/get-flash', (req, res) => {
      const messages = req.flash('info');
      res.send(`Flash messages: ${messages}`);
    });
    ```
5. Start your Express server and test the flash message functionality:
    ```javascript
    app.listen(3000, () => {
      console.log('Server is running on port 3000');
    });
    ```
Note: Flash messages are typically used to display temporary messages to users, such as success or error notifications, and are cleared after being displayed once.

Passport:
- Passport is an authentication middleware for Node.js that provides a simple and flexible way to authenticate users in your application. It supports various authentication strategies, such as local authentication, OAuth, and more. Passport allows you to easily integrate authentication into your Express applications and manage user sessions.

Steps to use Passport with local strategy:
1. Install the necessary packages:
    ```
    npm install passport passport-local express-session
    ``` 
2. Import the required modules in your application:
    ```javascript
const passport = require('passport');
const LocalStrategy = require('passport-local').Strategy;
const User = require('./models/user'); // Assuming you have a User model for authentication
    ```
3. Now we require sessions as a middleware to use passport sessions. 
4. Assuming you have used session as a middleware, you can initialize passport and use passport sessions:
    ```javascript
app.use(passport.initialize());
app.use(passport.session());
    ```
-Here passport.session() is used to persist login sessions. It will serialize the user instance to the session and deserialize it on subsequent requests.
In simple words, passport.initialize() is used to initialize Passport and set up the necessary middleware for authentication, while passport.session() is used to manage user sessions and keep users logged in across multiple requests.

5. Configure the local strategy for authentication:
    ```javascript
passport.use(new LocalStrategy(User.authenticate()));
    ```
- Here, we are using the LocalStrategy provided by Passport and passing the authenticate method from our User model. This method is responsible for verifying the user's credentials (username and password) during the authentication process.
- In simple words, the LocalStrategy is a strategy for authenticating users using a username and password. By passing User.authenticate() to the LocalStrategy, we are telling Passport to use the authentication method defined in our User model to verify the user's credentials when they attempt to log in.

6. Set up serialization and deserialization of user instances:
    ```javascript
passport.serializeUser(User.serializeUser());
passport.deserializeUser(User.deserializeUser());
    ```
- Serialization is the process of converting a user instance into a format that can be stored in the session (usually a user ID), while deserialization is the process of converting the stored session data back into a user instance. By using User.serializeUser() and User.deserializeUser(), we are telling Passport how to serialize and deserialize user instances based on the methods defined in our User model.
- In simple words, serialization and deserialization are used to manage user sessions. When a user logs in, their user instance is serialized and stored in the session. When the user makes subsequent requests, the session data is deserialized to retrieve the user instance, allowing Passport to maintain the user's authentication state across requests.
